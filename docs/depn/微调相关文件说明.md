# DEPN项目微调相关Python文件详细说明

## 一、核心微调文件

### 1. `data/run_clm_no_trainer.py` ⭐ **核心微调脚本**
**作用**：Llama3-8B模型的因果语言模型（CLM）微调主脚本

**主要功能**：
- 基于HuggingFace Transformers实现，适配DEPN项目
- 支持从预训练模型进行微调（Causal Language Modeling）
- 支持8-bit量化、梯度检查点等内存优化技术
- 支持断点续训（checkpoint resume）
- 支持分布式训练（通过Accelerate）
- 自动处理数据预处理、tokenization、训练循环、评估等

**关键参数**：
- `--model_name_or_path`: 预训练模型路径
- `--train_file` / `--validation_file`: 训练/验证数据文件
- `--num_train_epochs`: 训练轮数
- `--learning_rate`: 学习率
- `--per_device_train_batch_size`: 每设备批次大小
- `--gradient_accumulation_steps`: 梯度累积步数
- `--checkpointing_steps`: 检查点保存频率
- `--load_in_8bit`: 8-bit量化
- `--gradient_checkpointing`: 梯度检查点（节省显存）

**调用方式**：
```bash
python accelerate_cli.py launch run_clm_no_trainer.py [参数...]
```

---

### 2. `data/eval_llama3_finetune.py` ⭐ **微调模型评估脚本**
**作用**：评估微调后的Llama3-8B模型性能

**主要功能**：
- 加载基础模型和微调模型进行对比
- 计算困惑度（Perplexity, PPL）指标
- 生成文本样本进行定性评估
- 支持中文和英文提示词测试

**关键函数**：
- `load_model_and_tokenizer()`: 加载模型和tokenizer（fp16精度）
- `build_eval_dataset()`: 构建评估数据集（固定长度块）
- `compute_perplexity()`: 计算困惑度
- `generate_samples()`: 生成文本样本

**使用场景**：
- 微调后评估模型性能
- 对比微调前后的效果
- 验证模型是否过拟合

---

## 二、数据预处理文件

### 3. `data/preprocess_enron.py` ⭐ **数据预处理脚本**
**作用**：预处理Enron邮件数据，生成训练/验证/测试集

**主要功能**：
- 从`maildir/`目录读取原始邮件文件
- 清理邮件头信息（From, Subject, Date等）
- 提取电话号码并格式化为空格分隔的数字（便于tokenization）
- 过滤邮箱地址
- 将数据分割为train/valid/test（最后3000条为valid，3000-6000为test，其余为train）
- 保存电话号码列表到`all_Tel.txt`

**输出文件**：
- `./temp_data/train.txt`: 训练集
- `./temp_data/valid.txt`: 验证集
- `./temp_data/test.txt`: 测试集
- `./all_Tel.txt`: 提取的电话号码列表

**关键处理**：
- 电话号码格式转换：`"8773155218"` → `"8 7 7 3 1 5 5 2 1 8"`
- 添加提示模板：`"My phone number is ***. "`等

## 三、辅助工具文件

### 4. `data/accelerate_cli.py` ⭐ **Accelerate CLI工具**
**作用**：HuggingFace Accelerate的命令行接口，用于启动分布式训练

**主要功能**：
- 封装Accelerate的launch命令
- 支持多GPU/多机分布式训练
- 自动处理设备分配和进程管理

**使用方式**：
```bash
python accelerate_cli.py launch [accelerate参数] [训练脚本] [脚本参数]
```

---

### 5. `data/get_memorization.py` 
**作用**：分析模型对隐私数据的记忆化程度

**主要功能**：
- 检测模型是否记忆了训练数据中的隐私信息（电话号码、姓名等）
- 计算隐私泄露风险指标（Exposure、MRR、PPL）
- 输出高风险记忆化的文本到文件

**隐私类型**：
- `TEL`: 电话号码（使用Exposure指标）
- `NAME`: 姓名（使用MRR指标）
- `RANDOM`: 随机文本（使用PPL指标）

**使用场景**：
- 微调后评估隐私泄露风险
- 确定需要保护的隐私数据范围

---

### 6. `data/mask_text2json.py`
**作用**：将文本数据转换为JSON格式

**主要功能**：
- 将采样后的隐私文本转换为JSON格式
- 用于后续的隐私神经元定位和编辑流程

---

### 7. `data/sample_target_privacy.py`
**作用**：从记忆化的隐私数据中采样目标隐私信息

**主要功能**：
- 从`memorized_TEL.txt`、`memorized_NAME.txt`等文件中随机采样
- 为后续的隐私神经元编辑提供目标数据

---

## 四、启动脚本（Shell脚本，调用Python文件）

### 8. `data/run_llama3.sh` ⭐ **标准微调启动脚本**
**作用**：启动Llama3-8B的标准微调流程

**配置特点**：
- 使用8-bit量化
- 梯度检查点
- 每5000步保存检查点
- 学习率：2e-5
- 训练5个epoch

**关键参数**：
```bash
--num_train_epochs 5
--checkpointing_steps 5000
--learning_rate 2e-5
--load_in_8bit
--gradient_checkpointing
```

---

### 9. `data/run_llama3_stable.sh` ⭐ **稳定版微调启动脚本**
**作用**：使用更稳定配置的微调启动脚本

**配置特点**：
- 更小的学习率（5e-6）
- 余弦学习率调度
- 梯度裁剪（max_grad_norm=1.0）
- 权重衰减（0.01）
- 更少的训练步数（10000步）

**使用场景**：
- 当标准微调出现NaN或训练不稳定时
- 需要更保守的训练策略时

---

## 五、微调流程总结

### 完整微调流程：

1. **数据预处理**
   ```bash
   python preprocess_enron.py
   ```
   - 生成 `temp_data/train.txt`, `temp_data/valid.txt`

2. **执行微调**
   ```bash
   bash run_llama3.sh
   # 或
   bash run_llama3_stable.sh
   ```
   - 调用 `accelerate_cli.py launch run_clm_no_trainer.py`
   - 输出模型到 `./model/llama3_8b_ep5/`

3. **评估微调模型**
   ```bash
   python eval_llama3_finetune.py
   ```
   - 计算困惑度
   - 生成文本样本

4. **分析隐私记忆化**（可选）
   ```bash
   python get_memorization.py [参数...]
   ```

## 八、注意事项

1. **内存优化**：Llama3-8B模型较大，建议使用：
   - `--load_in_8bit`: 8-bit量化
   - `--gradient_checkpointing`: 梯度检查点
   - `--low_cpu_mem_usage`: 低CPU内存使用

2. **检查点保存**：建议设置`--checkpointing_steps`定期保存，防止训练中断

3. **数据格式**：确保训练数据是纯文本格式（.txt），每行一个样本

4. **模型路径**：确保预训练模型路径正确，可以是本地路径或HuggingFace模型ID
