#### 1.1 Batch 处理改进（第290-295行）

**分析**：这个修改是正确的。在计算积分梯度时，需要将输入重复 batch_size 次，这样可以并行处理多个插值点。

#### 1.2 `tgt_pos = len(input_ids) - 2` 的修改（第42行）

**分析**：这个修改是**合理的**，但需要理解其原理：
- 在 Causal LM 中，如果我们要预测位置 `i` 的下一个token，我们需要在位置 `i` 提取 hidden state
- `tgt_pos = len(input_ids) - 2` 意味着在倒数第二个位置提取 hidden state
- 模型会预测 `tgt_pos + 1`（也就是最后一个位置）的 token
- 这符合 Causal LM 的预测机制

### ⚠️ 需要注意的问题

#### 1.3 Padding 后的位置问题（第53-54行）

**潜在问题**：
```python
if target_token_id is None and len(input_ids) >= 2:
    target_token_id = input_ids[tgt_pos + 1]
```

**问题分析**：
- 在 padding 之前，`input_ids` 的长度是实际token数量
- `tgt_pos = len(input_ids) - 2` 是相对于**未padding**的序列计算的
- 但在第56-58行，`input_ids` 会被 padding 到 `max_seq_length`
- 所以 `tgt_pos + 1` 可能会指向 padding token

**建议修复**：
```python
if target_token_id is None and len(input_ids) >= 2:
    # 使用未padding前的长度
    actual_len = len(tokenized["input_ids"])
    if tgt_pos + 1 < actual_len:
        target_token_id = input_ids[tgt_pos + 1]
    else:
        # fallback: 使用最后一个实际token
        target_token_id = input_ids[actual_len - 1]
```

或者更简单的方法，在 padding 之前就确定 `target_token_id`：
```python
# 在 padding 之前确定 target_token_id
if target_token_id is None and len(input_ids) >= 2:
    target_token_id = input_ids[tgt_pos + 1]  # 此时 input_ids 还未padding

# 然后再 padding
padding_length = max_seq_length - len(input_ids)
```

## 2. `3_edit_privacy_neurons_llama.py` 的修改分析

### ⚠️ 需要注意的问题

#### 2.1 `get_exposure_llama` 中的位置计算（第159-163行）

**当前代码**：
```python
prefix_len = len(tokenizer.encode(scaled_input_texts[i], add_special_tokens=False))
target_pos = max(prefix_len - 1, 0)
if target_pos >= outputs.logits[i].shape[0]:
    continue
logits = outputs.logits[i][target_pos]  # Logits at target position
```

**问题分析**：
1. **重复编码问题**：`scaled_input_texts[i]` 已经在模型 forward 时被编码了，这里再次编码可能不一致
2. **位置计算可能不准确**：`prefix_len - 1` 可能不是正确的预测位置
3. **与 `1_calculate_attribution_llama.py` 的逻辑不一致**：那里使用 `tgt_pos = len(input_ids) - 2`

**建议修复**：
```python
def get_exposure_llama(gold_text, scaled_input_texts, outputs, nums_encode, TOTAL_CANDIDATES, tokenizer):
    """
    Calculate canary exposure for phone numbers using Llama3.
    For Causal LM, we predict next token instead of masked token.
    """
    secret_nums = ''.join(re.findall(r'\d+', gold_text))
    rank = 1
    
    for i in range(len(secret_nums)):
        if i >= len(outputs.logits):
            continue
        
        # 获取实际的序列长度（从 attention_mask 或 input_ids）
        # 注意：outputs.logits 的形状是 (batch_size, seq_len, vocab_size)
        # 对于 Causal LM，位置 j 的 logits 预测的是位置 j+1 的 token
        
        # 方法1: 使用最后一个位置的 logits（推荐）
        # 因为 scaled_input_texts[i] 的最后一个位置应该预测下一个token
        seq_len = outputs.logits[i].shape[0]
        if seq_len == 0:
            continue
        
        # 使用最后一个位置（seq_len - 1）的 logits
        # 这个 logits 预测的是下一个token（也就是我们要找的数字）
        target_pos = seq_len - 1
        logits = outputs.logits[i][target_pos]
        
        rank += get_tar_rank(secret_nums[i], logits, nums_encode) * pow(10, len(secret_nums) - 1 - i)
    
    canary_exposure = math.log(TOTAL_CANDIDATES, 2) - math.log(rank, 2)
    return rank, canary_exposure
```

**或者，更准确的方法**：
```python
def get_exposure_llama(gold_text, scaled_input_texts, outputs, nums_encode, TOTAL_CANDIDATES, tokenizer):
    """
    Calculate canary exposure for phone numbers using Llama3.
    For Causal LM, we predict next token instead of masked token.
    """
    secret_nums = ''.join(re.findall(r'\d+', gold_text))
    rank = 1
    
    for i in range(len(secret_nums)):
        if i >= len(outputs.logits):
            continue
        
        # 计算当前前缀的实际长度（不包括padding）
        # 注意：在 Causal LM 中，位置 j 的 logits 预测位置 j+1 的token
        # 所以如果输入长度是 L（不包括padding），最后一个有效位置的 logits（位置 L-1）
        # 预测的是第 L 个token（即我们要预测的数字）
        
        # 获取输入的实际长度
        # 假设我们传入的 inputs 包含了 attention_mask，可以从中获取
        # 或者从 logits 的形状推断
        
        # 简单方法：使用最后一个位置的 logits
        seq_len = outputs.logits[i].shape[0]
        if seq_len == 0:
            continue
        
        # 最后一个位置的 logits 预测下一个token
        target_pos = seq_len - 1
        logits = outputs.logits[i][target_pos]
        
        rank += get_tar_rank(secret_nums[i], logits, nums_encode) * pow(10, len(secret_nums) - 1 - i)
    
    canary_exposure = math.log(TOTAL_CANDIDATES, 2) - math.log(rank, 2)
    return rank, canary_exposure
```

2. ✅ `tgt_pos = len(input_ids) - 2` 的修改（符合 Causal LM 机制）

2. **`3_edit_privacy_neurons_llama.py` 第159-163行**：位置计算逻辑需要改进，使用最后一个位置的 logits 更可靠

### 建议的修复

我已经在上面提供了具体的修复建议。主要原则是：
1. 在 Causal LM 中，位置 `i` 的 logits 预测的是位置 `i+1` 的 token
2. 对于序列长度为 L 的输入，位置 `L-1` 的 logits 预测下一个token
3. 需要确保位置索引不超出实际token序列的范围（排除padding）
